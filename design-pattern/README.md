# 设计模式学习记录

个人学习设计模式记录项目。

面向对象设计中的六大经典原则（SOLID + 迪米特法则）是构建可维护、可扩展软件系统的基石：

| 原则 | 全称 | 核心思想 |
|------|------|----------|
| SRP | 单一职责原则 | 一个类只负责一件事 |
| OCP | 开闭原则 | 对扩展开放，对修改关闭 |
| LSP | 里氏替换原则 | 子类可以替换父类 |
| ISP | 接口隔离原则 | 接口应细粒度，不强迫依赖不需要的方法 |
| DIP | 依赖倒置原则 | 依赖抽象，不依赖具体实现 |
| LoD | 迪米特法则 | 只与直接朋友通信，降低耦合 |

为了满足设计的六大原则，软件行业总结了日常开发中常用的 23 种设计模式，**原则指导模式，模式体现原则**。

---

## 一、创建型模式（Creational Patterns）

关注对象的创建方式，将对象的创建与使用分离。

### 1. 单例模式（Singleton）
**包路径：** `singleton/`

保证一个类只有一个实例，并提供全局访问点。

| 实现方式 | 类名 | 说明 |
|----------|------|------|
| 饿汉式 | `Eager` | 类加载时即创建实例，线程安全 |
| 双重检查锁 | `DoubleCheckLocking` | 懒加载 + `volatile` 保证可见性，推荐使用 |
| 静态内部类 | `InnerClass` | 懒加载，利用类加载机制保证线程安全 |

```java
// 推荐：双重检查锁
DoubleCheckLocking instance = DoubleCheckLocking.getInstance();
```

---

### 2. 工厂模式（Factory）
**包路径：** `factory/`

封装对象的创建逻辑，调用方无需关心具体实现类。

#### 2.1 简单工厂（Simple Factory）
`factory/simple/` — 由一个工厂类根据参数决定创建哪种产品，违反开闭原则，适合产品类型固定的场景。

#### 2.2 工厂方法模式（Factory Method）
`factory/method/` — 定义创建对象的接口，由子类决定实例化哪个类，符合开闭原则。

```
ComputerFactory（接口）
├── HuaweiComputerFactory
└── XiaomiComputerFactory
```

#### 2.3 抽象工厂模式（Abstract Factory）
`factory/devise/` — 提供一个创建一系列相关产品的接口，无需指定具体类。

```
AbstractCompanyFactory（接口）
├── HuaweiCompanyFactory  →  HuaweiPhone
└── XiaomiCompanyFactory  →  XiaomiPhone
```

---

### 3. 建造者模式（Builder）
**包路径：** `builder/`

将复杂对象的构建过程与其表示分离，使同样的构建过程可以创建不同的表示。

```
ComputerBuilder（抽象建造者）
├── HuaweiComputerBuilder
└── XiaomiComputerBuilder
ComputerDirector（指挥者）  →  Computer（产品）
```

## 二、结构型模式（Structural Patterns）

关注类或对象的组合方式，形成更大的结构。

### 5. 适配器模式（Adapter）
**包路径：** `adapter/`

将一个类的接口转换成客户期望的另一个接口，使原本接口不兼容的类可以协同工作。

```
Adaptee（被适配者，已有接口）
ClassAdapter extends Adaptee implements Target  →  Target（目标接口）
```

```java
Target target = new ClassAdapter();
target.request();  // 内部调用 Adaptee.specificRequest()
```

## 三、行为型模式（Behavioral Patterns）

关注对象之间的职责划分与通信方式。

### 12. 策略模式（Strategy）
**包路径：** `strategy/`

定义一系列算法，封装每个算法，使它们可以互相替换，让算法的变化独立于使用算法的客户。

```
Strategy（接口）
├── AddOperation
└── SubtractOperation
Context（上下文，持有 Strategy 引用）
```

```java
Context context = new Context(new AddOperation());
context.execute(10, 5);  // 15
```

---

### 13. 模板方法模式（Template Method）
**包路径：** `template/`

在父类中定义算法的骨架，将某些步骤延迟到子类实现，不改变算法结构。

```
Game（抽象模板）
├── Basketball
└── Football
```
